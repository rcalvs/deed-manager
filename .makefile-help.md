# Makefile - Guia de Uso

Este Makefile simplifica o processo de build e release do Wurm Manager.

## Uso Básico

### Verificar versão atual
```bash
make check-version
```

### Atualizar apenas a versão
```bash
make update-version VERSION=0.2.2
# ou
make update-version 0.2.2
```

### Build completo (atualiza versão + build + prepara release)
```bash
make build VERSION=0.2.2
# ou
make build 0.2.2
```

### Build para plataforma específica
```bash
make build-windows    # Windows
make build-darwin     # macOS
make build-linux      # Linux
make build-all        # Todas as plataformas
```

### Release completo (build + tag + GitHub)
```bash
make release VERSION=0.2.2
# ou
make release 0.2.2
```

**Nota:** Requer GitHub CLI (`gh`) instalado para criar release automaticamente.

### Outros comandos úteis
```bash
make dev              # Modo desenvolvimento
make clean            # Limpa arquivos de build
make install-deps     # Instala dependências do frontend
make help             # Mostra ajuda completa
```

## Exemplos Práticos

### Criar um novo release v0.2.2

```bash
# Opção 1: Tudo de uma vez (recomendado)
make release 0.2.2

# Opção 2: Passo a passo
make update-version 0.2.2
make build-all
make prepare-release
# Depois criar release manualmente no GitHub
```

### Apenas fazer build sem atualizar versão

```bash
make build-windows
# ou
make build-all
```

### Preparar binários para release manual

```bash
make build-all
make prepare-release
# Os binários estarão em release/
```

## Estrutura de Comandos

O Makefile suporta duas formas de passar a versão:

1. **Variável de ambiente**: `make build VERSION=0.2.2`
2. **Argumento posicional**: `make build 0.2.2`

Ambas funcionam da mesma forma!
